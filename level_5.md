# Bandit Level 5 → Level 6

## 目標

`inhere` ディレクトリ以下にある大量のファイルの中から、以下の条件をすべて満たすファイルを見つけてパスワードを入手する。

- 人間が読める（ASCII テキスト）
- サイズが **1033 バイト**
- 実行可能ファイルではない

## 使うコマンド

- `ssh`  : サーバーへのリモートログイン
- `cd`   : ディレクトリの移動
- `find` : 条件を指定してファイルを検索する
- `cat`  : ファイルの内容を表示

## 手順

### Step 1: SSH でログイン

```bash
ssh bandit5@bandit.labs.overthewire.org -p 2220
```

パスワードは Level 4 で入手したもの（`passwords.txt` の `bandit5` 行）を使う。

### Step 2: inhere ディレクトリに移動

```bash
cd inhere/
```

`ll` で確認すると大量のサブディレクトリが並んでいる。手作業で探すのは現実的でない。

### Step 3: 条件を指定してファイルを検索

```bash
find . -type f -size 1033c ! -executable
```

マッチしたファイルのパスが表示される（例: `./maybehere07/.file2`）。

### Step 4: ファイルを読む

```bash
cat ./maybehere07/.file2
```

（表示されたパスに合わせて変える）

表示された文字列が **bandit6 のパスワード**。

## ポイント

### `find` コマンドの基本構造

```
find <検索場所> <条件> <条件> ...
```

今回使ったオプションの意味：

| オプション       | 意味                                   |
|------------------|----------------------------------------|
| `.`              | カレントディレクトリ以下を再帰的に検索 |
| `-type f`        | ファイルのみ対象（ディレクトリを除く） |
| `-size 1033c`    | サイズが 1033 バイト（`c` = bytes）    |
| `! -executable`  | 実行可能ではないもの（`!` は否定）     |

---

### `find` はサブディレクトリも自動で検索する

`find` は指定した場所の **すべてのサブディレクトリ** を自動でたどって検索する。
`inhere` の中にさらにディレクトリが入れ子になっていても一発で探せる。

```bash
# ls はカレントの1階層しか見えない
ls

# find は何階層深くても全部探してくれる
find . -type f -size 1033c ! -executable
```

---

### サイズ指定の単位

`find` のサイズ指定には単位が必要。よく使うものは以下の通り：

| 単位 | 意味          |
|------|---------------|
| `c`  | バイト        |
| `k`  | キロバイト    |
| `M`  | メガバイト    |

今回は `1033c`（1033 バイト）を指定した。

## 結果

入手したパスワードを `passwords.txt` の `bandit6` 行に記録する。
