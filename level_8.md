# Bandit Level 8 → Level 9

## 目標

`data.txt` に大量の行が並んでいる。その中で **1回しか出現しない行** がパスワード。

## 使うコマンド

- `ssh`   : サーバーへのリモートログイン
- `sort`  : 行をアルファベット順に並び替える
- `uniq`  : 重複した行をまとめる
- `|`     : コマンドの出力を次のコマンドに渡す（パイプ）

## 手順

### Step 1: SSH でログイン

```bash
ssh bandit8@bandit.labs.overthewire.org -p 2220
```

パスワードは Level 7 で入手したもの（`passwords.txt` の `bandit8` 行）を使う。

### Step 2: 1回しか出現しない行を探す

```bash
sort data.txt | uniq -u
```

1行だけ表示される。それが **bandit9 のパスワード**。

## ポイント

### なぜ `sort` が必要なのか？

`uniq` は **隣り合う行** の重複しか検出できない。
そのまま `uniq -u data.txt` を実行しても正しく動かない。

```
AAA   ← 1回目
BBB   ← uniq はここで AAA との重複を検知できない
AAA   ← 2回目（離れているので uniq は重複と判定しない）
```

`sort` で先に並び替えると、同じ内容の行が必ず隣り合うようになり、
`uniq` が正しく重複を検出できる。

```
AAA   ← 並び替え後：同じ行が隣り合う
AAA
BBB
```

---

### `uniq -u` の `-u` とは？

`-u` は "unique"（一意）の意味。重複している行をすべて除外して、
**1回しか出現しない行だけ** を表示するオプション。

| オプション | 意味                                 |
|------------|--------------------------------------|
| （なし）   | 重複をまとめて1行にして表示          |
| `-u`       | 1回しか出現しない行だけ表示          |
| `-d`       | 重複している行だけ表示               |
| `-c`       | 各行が何回出現するか数を表示         |

---

### パイプ `|` とは？

`|` は左のコマンドの出力を右のコマンドの入力として渡す記号。

```
sort data.txt | uniq -u
      ↑                ↑
  まず並び替え    次に重複を処理
```

コマンドをパイプでつなぐことで、複数の処理を1行で書ける。
Linux では `|` を使ったコマンドの組み合わせが非常に多用される。

## 結果

入手したパスワードを `passwords.txt` の `bandit9` 行に記録する。
