# Bandit Level 2 → Level 3

## 目標

ホームディレクトリにある `--spaces in this filename--` という名前のファイルを読んで、
Level 3 のパスワードを入手する。

## 使うコマンド

- `ssh` : サーバーへのリモートログイン
- `ls`  : ファイル一覧の確認
- `cat` : ファイルの内容を表示

## 手順

### Step 1: SSH でログイン

```bash
ssh bandit2@bandit.labs.overthewire.org -p 2220
```

パスワードは Level 1 で入手したもの（`passwords.txt` の `bandit2` 行）を使う。

### Step 2: ホームディレクトリを確認

```bash
ls
```

`--spaces in this filename--` という名前のファイルがある。

### Step 3: ファイルを読む

`./` をつけてパスとして指定する方法が確実。

```bash
cat "./ --spaces in this filename--"
```

実際には以下のように入力する（`./` のあとにスペースはない）。

```bash
cat "./--spaces in this filename--"
```

表示された文字列が **bandit3 のパスワード**。

## ポイント

このレベルには **2つの罠** がある。

---

### 罠1: スペースを含むファイル名

シェルはスペースを「コマンドの区切り」として解釈する。
そのままだと複数のファイル名として誤解釈されてしまう。

```bash
# NG: スペースで区切られ複数ファイルとして解釈される
cat --spaces in this filename--
```

→ クォートで囲むことで1つのファイル名として渡せる。

---

### 罠2: `--` で始まるファイル名

多くのコマンドでは `--` を「オプション指定の終わり」を示す特別な記号として扱う。
そのため `cat "--spaces in this filename--"` と書いても、
`--spaces` がオプションとして誤解釈されてエラーになる可能性がある。

```bash
# NG（エラーになる場合がある）: --spaces がオプションとして解釈される
cat "--spaces in this filename--"
```

---

### 解決策: `./` でパスとして明示する

`./` を先頭につけることで「カレントディレクトリにあるファイル」とシェルに伝わり、
`--` も含めてファイル名として正しく扱われる。

```bash
# OK: ./で始まるのでオプションではなくパスとして認識される
cat "./--spaces in this filename--"
```

---

### 別の解決策: `--` でオプション解析を終了させる

`cat` に渡す引数の前に単独の `--` を置くと「以降はオプションではなくファイル名」と伝えられる。

```bash
cat -- "--spaces in this filename--"
```

---

### まとめ: ファイル名に仕掛けられた罠の対処法

| 罠                    | 対処法                              |
|-----------------------|-------------------------------------|
| スペースを含む        | クォートで囲む or バックスラッシュ  |
| `--` や `-` で始まる  | `./` をつける or `--` を前に置く    |

## 結果

入手したパスワードを `passwords.txt` の `bandit3` 行に記録する。
