# Bandit Level 1 → Level 2

## 目標

ホームディレクトリにある `-` という名前のファイルを読んで、Level 2 のパスワードを入手する。

## 使うコマンド

- `ssh` : サーバーへのリモートログイン
- `ls`  : ファイル一覧の確認
- `cat` : ファイルの内容を表示

## 手順

### Step 1: SSH でログイン

```bash
ssh bandit1@bandit.labs.overthewire.org -p 2220
```

パスワードは Level 0 で入手したもの（`passwords.txt` の `bandit1` 行）を使う。

### Step 2: ホームディレクトリを確認

```bash
ls
```

`-` という名前のファイルがある。

### Step 3: ファイルを読む

```bash
cat ./-
```

表示された文字列が **bandit2 のパスワード**。

## ポイント

### なぜ `cat -` ではダメなのか？

シェルでは `-` は「標準入力（キーボード入力）」を意味する特別な記号として解釈される。
そのため `cat -` と入力すると、ファイルを開かずにキーボード入力待ちになってしまう。

### `./` をつけると何が変わるのか？

`./` は「今いるディレクトリの中の」という意味のパス指定。
`cat ./-` とすることで「`-` という名前のファイル」と明示でき、シェルが特別扱いしなくなる。

```bash
# NG: 標準入力待ちになってフリーズする
cat -

# OK: カレントディレクトリの - ファイルを開く
cat ./-
```

`Ctrl + C` でフリーズから抜け出せる。

### 別の書き方

リダイレクト `<` を使っても同じ結果が得られる。

```bash
cat < -
```

## 結果

入手したパスワードを `passwords.txt` の `bandit2` 行に記録する。
