# Bandit Level 3 → Level 4

## 目標

`inhere` ディレクトリの中にある隠しファイルを読んで、Level 4 のパスワードを入手する。

## 使うコマンド

- `ssh`  : サーバーへのリモートログイン
- `ls`   : ファイル一覧の確認
- `ll`   : 隠しファイルを含めた詳細一覧（`ls -la` のエイリアス）
- `cd`   : ディレクトリの移動
- `cat`  : ファイルの内容を表示

## 手順

### Step 1: SSH でログイン

```bash
ssh bandit3@bandit.labs.overthewire.org -p 2220
```

パスワードは Level 2 で入手したもの（`passwords.txt` の `bandit3` 行）を使う。

### Step 2: ホームディレクトリを確認

```bash
ls
```

`inhere` というディレクトリがある。

### Step 3: inhere ディレクトリに移動

```bash
cd inhere/
```

### Step 4: 隠しファイルを探す

```bash
ll
```

`.` で始まるファイルが見つかる（例: `...Hiding-From-You`）。

### Step 5: ファイルを読む

```bash
cat ./...Hiding-From-You
```

ファイル名が異なる場合は `ll` で確認した名前を使う。
表示された文字列が **bandit4 のパスワード**。

## ポイント

### 隠しファイルとは？

Linux では、ファイル名が `.`（ドット）で始まるファイルを **隠しファイル** と呼ぶ。
設定ファイルなどに使われており、通常の `ls` では表示されない。

```bash
# NG: 隠しファイルは表示されない
ls

# OK: 隠しファイルも含めて詳細表示される
ll
```

`ll` は `ls -la` のエイリアス（短縮コマンド）。`-a` は "all"（すべて）、`-l` は詳細表示の意味。

---

### `ls -a` の出力に必ず出てくる `.` と `..`

```
.   ..   ...Hiding-From-You
```

| 表示 | 意味                           |
|------|--------------------------------|
| `.`  | カレントディレクトリ自身       |
| `..` | 1つ上のディレクトリ（親）      |

これらはファイルではなくディレクトリへの参照なので無視してよい。
パスワードが入っているのはそれ以外の `.` で始まるファイル。

---

### `ll` の出力例

```
drwxr-xr-x  2 bandit3 bandit3 4096 Apr  3 15:17 ./
drwxr-xr-x 70 root    root    4096 Apr  3 15:20 ../
-rw-r-----  1 bandit4 bandit3   33 Apr  3 15:17 ...Hiding-From-You
```

パーミッション・オーナー・サイズ・更新日時も一緒に確認できる。

## 結果

入手したパスワードを `passwords.txt` の `bandit4` 行に記録する。
