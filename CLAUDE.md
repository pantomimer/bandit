# CLAUDE.md — Bandit 攻略ノート プロジェクト概要

## プロジェクトの目的

OverTheWire Bandit (https://overthewire.org/wargames/bandit/) の
全レベル(Level 0 → Level 34)を解説する教育用 Markdown ドキュメントを作成する。

作成者はプログラミング講師。受講者が自力で理解できるよう、
「なぜそのコマンドを使うのか」「何が起きているのか」を丁寧に書くことを重視する。

---

## リポジトリ構成

```
bandit/
├── CLAUDE.md          # このファイル（プロジェクト概要）
├── README.md          # 全体進捗・接続情報
├── passwords.txt      # 各レベルのパスワード記録（非公開扱い）
├── level_0.md         # Level 0 → 1 の解説（テンプレート兼用）
├── level_1.md         # Level 1 → 2 の解説
├── level_2.md         # ...
└── ...
```

各レベルのファイル名は `level_<N>.md`（N = 出発レベル番号）とする。

---

## 各レベルファイルのフォーマット

`level_0.md` を標準テンプレートとして使う。構成は以下の通り：

```markdown
# Bandit Level N → Level N+1

## 目標
（何をすれば次のレベルに進めるか、1〜2文で）

## 使うコマンド
（箇条書き。コマンド名 : 一行説明）

## 手順
（Step 1, Step 2, ... の形式。コードブロック付き）

## ポイント
（なぜそうするか／ハマりやすい点をわかりやすく説明）

## 結果
（入手パスワードを passwords.txt に記録する旨）
```

---

## 執筆スタイルのルール

- **対象読者**: Linux コマンドに慣れていない初学者
- **コマンドは必ずコードブロック**で示す（インラインコードより見やすいため）
- **オプションの意味**を都度説明する（例: `-p 2220` は「ポート番号の指定」）
- 「なぜこのコマンドか」を先に書き、「どう使うか」をあとに書く順番を意識する
- パスワード文字列は本文中に書かず、`passwords.txt` への記録指示にとどめる

## 接続情報（毎回共通）

```
ホスト : bandit.labs.overthewire.org
ポート : 2220
ユーザー: bandit<レベル番号>  例) bandit1, bandit2, ...
```

## Claude への依頼パターン

| 依頼                            | 期待する出力                                  |
|---------------------------------|-----------------------------------------------|
| `level_N.md を作って`           | フォーマットに沿った解説 Markdown を新規作成  |
| `level_N.md を改善して`         | 既存ファイルを読み、わかりやすさを向上        |
| `level_N のポイントを説明して`  | 該当レベルで学べる技術概念をわかりやすく解説  |

## 注意事項

- passwords.txt は学習記録用。パスワードを外部に公開しない。
- サーバーへの攻撃的な行為は行わない。
- 解説は OverTheWire の利用規約の範囲内で作成する。
